Преврати записанный playwright codegen-скрипт в полноценный слоёный тест по
конвенциям проекта (см. CLAUDE.md).

Целевой файл: `$ARGUMENTS` (путь к скрипту в `recorded_scripts/`).

Выполни строго по порядку:

1. **Прочитай** записанный скрипт. Выпиши все действия и использованные локаторы.

2. **Найди реальные селекторы.** Codegen часто пишет нестабильные локаторы
   (длинные тексты, сессионные URL). Если селекторы вызывают сомнение — напиши
   временный debug-скрипт, который логинится и дампит нужный участок DOM
   (`page.evaluate`) и/или делает скриншот. Запусти, определи устойчивые селекторы,
   затем **удали debug-скрипт и его артефакты** (png/html).

3. **PageObjects (`pages/`)** — на каждую логическую страницу свой класс.
   Только локаторы как `@property` → `Locator`. Никаких действий. Обнови
   `pages/__init__.py`.

4. **Steps (`steps/`)** — класс `*Steps(page)`. Сюда все действия, ожидания и
   `expect`-проверки. Атомарные шаги + при необходимости композитный. Применяй
   Siebel-правила: никакого `networkidle`, жди `#maskoverlay` hidden,
   `click(no_wait_after=True)` на медленных навигациях, `expect.set_options(timeout=...)`.
   Обнови `steps/__init__.py`.

5. **Test data (`test_data/`)** — если нужны учётки/данные, добавь их в реестр по
   алиасу (логин в данных, секреты из `.env`).

6. **Test (`tests/e2e/`)** — обращается ТОЛЬКО к методам шагов. Ни локаторов, ни
   `settings`, ни `page` API напрямую.

7. **Запусти** `python -m pytest <new-test> -v -s` и чини под Siebel, пока не пройдёт.
   Тесты медленные (1-2 мин) — запускай в фоне и дожидайся.

В конце покажи список созданных/изменённых файлов и итог прогона.
