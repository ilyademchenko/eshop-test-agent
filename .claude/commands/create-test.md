## Этап 1 — Аудит готовности

Определи, какие страницы, PageObject'ы и шаги нужны для сценария.
Затем проверь, что уже есть в проекте.

### 1.1 Декомпозиция сценария

Выпиши список страниц/экранов, которые участвуют в сценарии, и для каждой:
- какие **локаторы** потребуются (секции, кнопки, поля ввода);
- какие **шаги** (действия + проверки) нужны;
- какие **тестовые данные** используются.

### 1.2 Инвентаризация проекта

Проверь:
- `pages/` — какие PageObject уже существуют; перечитай каждый нужный файл и
  выпиши доступные `@property`.
- `steps/` — какие `*Steps`-классы есть; перечитай каждый нужный и выпиши методы.
- `test_data/` — есть ли нужные пользователи/данные.

### 1.3 Отчёт о пробелах

Составь таблицу:

| Компонент | Нужен | Есть | Пробел |
|---|---|---|---|
| `pages/some_page.py` | да | нет | создать + наполнить локаторами |
| `SomeSteps.do_action()` | да | нет | реализовать |
| `test_data/products.py` | да | да | — |

Если пробелов нет — переходи сразу к Этапу 3.
Если есть — переходи к Этапу 2.

## Этап 2 — Устранение пробелов

Для каждого недостающего PageObject выполни **`/scan-page <PageName или URL>`**.

`/scan-page` откроет страницу в браузере, снимет DOM и наполнит PageObject
реальными локаторами — не пропускай этот шаг и не придумывай локаторы вручную.

Порядок:
1. Запусти `/scan-page` для каждой страницы с пробелом.
2. После каждого `/scan-page` убедись, что нужные локаторы появились в PageObject.
3. Если нужные данные отсутствуют в `test_data/` — добавь их (логин в данных,
   секреты из `.env`; см. структуру `test_data/users.py`).

## Этап 3 — Разработка теста

Создавай слои в порядке зависимостей: `pages → steps → test_data → tests`.

### 3.1 Steps (`steps/`)

Создай или дополни класс `*Steps`:
- Конструктор принимает `page: Page`, инстанцирует нужные PageObject'ы,
  вызывает `page.set_default_timeout(settings.NAVIGATION_TIMEOUT)`.
- В начале каждого файла: `expect.set_options(timeout=settings.NAVIGATION_TIMEOUT)`.
- Обязателен метод `wait_spinner_gone()` (ждёт `#maskoverlay` hidden, глотает исключения).
- **Siebel-правила** (нарушение = нерабочий тест):
  - Никакого `networkidle` — жди конкретные элементы DOM.
  - Перед любым кликом внутри кабинета — `await wait_spinner_gone()`.
  - Медленные навигации (вход, выход, переходы между view) → `click(no_wait_after=True)`,
    затем ждёт появления следующего якорного элемента.
  - `expect(locator).to_be_visible()` — всегда с `timeout=settings.NAVIGATION_TIMEOUT`.
- Имена методов — глагол: `open_page`, `fill_form`, `click_button`, `assert_result`.
- Докстроки на русском.
- Зарегистрируй класс в `steps/__init__.py`.

### 3.2 Test data (`test_data/`)

Добавь только то, что реально нужно тесту:
- Новые пользователи — в `test_data/users.py` (логин в коде, пароль через `.env`).
- Прочие данные (коды товаров, суммы) — в отдельный модуль `test_data/<entity>.py`.
- Зарегистрируй экспорт в `test_data/__init__.py`.

### 3.3 Test (`tests/e2e/`)

Файл `tests/e2e/test_<scenario_name>.py`:
- Класс `Test<ScenarioName>` с `@pytest.mark.asyncio`.
- Методы `async def test_*` принимают фикстуру `page` или `authenticated_page`.
- Тест **обращается ИСКЛЮЧИТЕЛЬНО к методам `*Steps`** — никакого прямого доступа
  к локаторам, `page` API, `settings` или `test_data`.
- Структура читается как пользовательский сценарий.

Используй `authenticated_page` если сценарий начинается уже после входа.
Используй `page` + `AuthSteps.login()` если вход — часть сценария.

## Этап 4 — Прогон и починка

1. Запусти тест **в фоне** (Siebel медленный, 1–2 мин):
   ```
   python -m pytest tests/e2e/test_<name>.py -v -s
   ```
2. При падении сопоставь с Siebel-граблями из `CLAUDE.md`:

   | Симптом | Вероятная причина | Правка |
   |---|---|---|
   | `TimeoutError` на `expect` за 5 с | не применён `expect.set_options(timeout=...)` | добавь в начало steps-файла |
   | Клик перехвачен оверлеем | не дождались `#maskoverlay` hidden | добавь `wait_spinner_gone()` перед кликом |
   | Элемент не найден / зависание навигации | нужен `no_wait_after=True` + ожидание следующего элемента | переписать шаг |
   | Дедлок pytest-asyncio | session-scoped async-фикстура | переведи в function-scope |
   | Элемент не найден после смены view | Siebel ещё рендерит контент | добавь явное ожидание якорного элемента |

3. Чини минимально — только ожидания и селекторы, не ослабляй проверки.
4. Повторяй до зелёного.

По завершении выведи:

**Созданные / изменённые файлы** (список с кратким описанием изменений).

**Итог прогона** — количество тестов, статус (passed/failed), время.

**Известные ограничения** — что тест не проверяет и почему
(например, «не проверяет обновление счётчика корзины — нужен scan-page страницы корзины»).
