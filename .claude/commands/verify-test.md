## Этап 1 — Анализ соответствия подходам

Прочитай тестовый файл и все связанные с ним слои (`steps/`, `pages/`, `test_data/`).
Проверь каждый файл по чеклисту.

### Чеклист: `tests/`

- [ ] Тест обращается **только** к методам `*Steps` — никакого прямого доступа к локаторам,
  `page` API, `settings` или `test_data`.
- [ ] Класс помечен `@pytest.mark.asyncio`.
- [ ] Метод(ы) — `async def test_*`.
- [ ] Фикстура: `authenticated_page` если вход уже выполнен; `page` + `AuthSteps.login()`
  если вход — часть сценария.
- [ ] Тест читается как пользовательский сценарий — без технических деталей.

### Чеклист: `steps/`

- [ ] Класс принимает `page: Page`, инстанцирует PageObject'ы, вызывает
  `page.set_default_timeout(settings.NAVIGATION_TIMEOUT)`.
- [ ] В начале файла: `expect.set_options(timeout=settings.NAVIGATION_TIMEOUT)`.
- [ ] Есть метод `wait_spinner_gone()` (или аналог из `AuthSteps`).
- [ ] **Нет** `wait_for_load_state("networkidle")`.
- [ ] Медленные навигации (вход, выход, смена view) — `click(no_wait_after=True)` + явное
  ожидание следующего якорного элемента.
- [ ] Перед кликами внутри кабинета — вызов `wait_spinner_gone()`.
- [ ] Все `expect(...)` вызываются с `timeout=settings.NAVIGATION_TIMEOUT` (или через
  `expect.set_options` в начале файла).
- [ ] Шаги — атомарные методы-глаголы (`open_`, `fill_`, `click_`, `assert_`).
- [ ] Докстроки на русском.

### Чеклист: `pages/`

- [ ] Только локаторы: `@property` → `Locator`. Никаких `click`/`fill`/`wait`/`expect`.
- [ ] Конструктор `__init__(self, page: Page)` сохраняет `self.page`.
- [ ] Нет сессионных id (`s_1_1_74_0` и т.п.) — только стабильные классы и роли.
- [ ] Класс зарегистрирован в `pages/__init__.py`.

### Чеклист: `test_data/`

- [ ] Пароли и секреты — только через `.env` (`settings.PASSWORD`), не захардкожены.
- [ ] Пользователи доступны через алиас (`get_user("КЗДТ")`), не напрямую.

### Отчёт

Составь таблицу нарушений:

| Файл | Нарушение | Правило из CLAUDE.md |
|---|---|---|
| `steps/cart_steps.py:34` | `wait_for_load_state("networkidle")` | Siebel-специфика п.1 |
| `tests/e2e/test_foo.py:12` | прямой вызов `page.locator(...)` | тест — только steps |

Если нарушений **нет** — сообщи об этом и переходи к Этапу 2.

Если нарушения **есть** — выведи таблицу и **запроси у пользователя разрешение**
на редактирование. Используй вопрос с вариантами:
- **Исправить всё** → применить все правки и перейти к Этапу 2.
- **Только критичные** → исправить только то, что мешает запуску (Siebel-правила),
  оставить остальное, перейти к Этапу 2.
- **Только запустить** → не трогать код, перейти сразу к Этапу 2.

Не вноси правки до получения ответа.

## Этап 2 — Запуск теста

Запусти тест **в фоне** (Siebel медленный, 1–2 мин на сценарий):

```
python -m pytest <цель> -v -s
```

Дождись завершения и получи полный вывод.

Если тест **прошёл** — сообщи результат и завершай.

Если тест **упал** — переходи к Этапу 3.

## Этап 3 — Починка

Сопоставь ошибку с Siebel-граблями и правь минимально.

| Симптом | Вероятная причина | Правка |
|---|---|---|
| `TimeoutError` на `expect` (5 с) | `expect.set_options(timeout=...)` не применён | добавь в начало steps-файла |
| Клик перехвачен оверлеем / `ElementClickInterceptedError` | спиннер `#maskoverlay` не ушёл | добавь `wait_spinner_gone()` перед кликом |
| `TimeoutError` на навигационный клик | нужен `no_wait_after=True` + ожидание следующего элемента | переписать шаг |
| Элемент не найден сразу после навигации | Siebel ещё рендерит — нет ожидания якорного элемента | жди `expect(anchor).to_be_visible()` |
| Дедлок pytest-asyncio / `fixture 'event_loop' not found` | session-scoped async-фикстура при function-scoped event loop | переведи в function-scope |
| Локатор устарел / не найден | сессионный id сменился или структура DOM изменилась | запусти `/scan-page <PageName>` и обнови локатор |

Порядок действий:
1. Определи корневую причину по трассировке.
2. Внеси минимальную правку в нужный слой (steps или pages — не трогай тест без необходимости).
3. Перезапусти.
4. Повторяй до зелёного или до исчерпания вариантов (не более 3 итераций).

Если за 3 итерации не удалось — не продолжай угадывать. Сообщи:
- точную ошибку и трассировку;
- что уже пробовал;
- гипотезу о причине;
- что нужно дополнительно (например: «требуется `/scan-page CartPage` — структура
  DOM корзины неизвестна»).
