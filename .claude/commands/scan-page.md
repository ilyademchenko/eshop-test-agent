Прочитай страницу и наполни соответствующий PageObject локаторами её секций и
интерактивных элементов. Цель действия: `$ARGUMENTS`.

Перед началом прочитай `CLAUDE.md` (архитектура слоёв, Siebel-специфика).

## Определение цели

`$ARGUMENTS` — это одно из трёх:
- **URL/путь** (начинается с `http` или `/`) — открой `settings.BASE_URL + путь`
  (или абсолютный URL). Определи или создай подходящий PageObject в `pages/`
  (имя по смыслу страницы, snake_case-файл + PascalCase-класс, зарегистрируй в
  `pages/__init__.py`).
- **Имя готового PageObject** (например `CatalogPage`) — найди его файл в `pages/`,
  определи URL/путь страницы из его докстроки/атрибутов или спроси, если неоднозначно,
  и открой её.
- **Суть/название страницы** (например `Каталог`, `Корзина`, `Заказы`) — когда прямого
  URL и готового PageObject нет, найди страницу по смыслу через навигацию (см. ниже).

## Поиск страницы по сути

Если цель задана смыслом (`Каталог` и т.п.), а не URL/PageObject:

1. **Найди путь к странице** через уже описанные локаторы и доступные действия
   пользователя: просмотри существующие PageObject (`GuestPage`, `MainPage`, ...) —
   там уже есть ссылки/кнопки навигации (`catalog_link`, пункты меню, попап профиля,
   вторичное меню). Выбери ссылку/действие, которое по смыслу ведёт на искомую страницу.
2. **Открой страницу** этим действием (логин при необходимости, см. Siebel-правила),
   определи, куда попал: URL, заголовок, ключевые блоки.
3. **СНАЧАЛА УТОЧНИ У ПОЛЬЗОВАТЕЛЯ** — до генерации любого кода. Дай краткое описание
   найденной страницы (как попал, URL/заголовок, что на ней видно, какой PageObject
   будет затронут/создан) и спроси, та ли это страница.
   - **Согласен** → переходи к «Шаги действия».
   - **Не согласен** → продолжи поиск похожей страницы (другая ссылка/действие/раздел),
     снова дай краткое описание и уточни ещё раз.
   - **Снова не согласен** (после второй попытки) → НЕ генерируй код. Предложи
     пользователю добавить страницу в код самостоятельно: подскажи имя/файл PageObject
     и как его зарегистрировать, при необходимости — известный URL-кандидат.

   Для уточнения используй вопрос с вариантами (да / искать дальше / добавлю сам).
   Не запускай скан DOM и не пиши локаторы, пока пользователь не подтвердил страницу.

## Шаги действия

1. **Открыть страницу.**
   - Запусти временный async-скрипт через Playwright. Если страница за авторизацией —
     переиспользуй шаги входа (`steps/auth_steps.py`, пользователь по умолчанию «КЗДТ»).
   - Соблюдай Siebel-правила: никакого `networkidle`; дождись `#maskoverlay` в `hidden`
     и появления якорного элемента контента; на медленных навигациях `no_wait_after=True`.
   - Сними DOM: пройди по элементам (`page.evaluate`), собери для каждого тег,
     устойчивый селектор, видимый текст/aria, роль, габариты. При необходимости —
     скриншот для ориентации. Временный скрипт и его артефакты (png/html) удали в конце.

2. **Определить секции страницы** — крупные структурные блоки (шапка, меню/навигация,
   сайдбар фильтров, основная область, карточки/списки, подвал, модалки).
   Для каждой секции добавь в PageObject локатор-`@property` → `Locator` с устойчивым
   селектором (предпочитай стабильные классы Siebel `.sib-ecustomer-*`, `get_by_role`,
   роли-ориентиры; избегай локаторов по длинному тексту и сессионным id).

3. **Определить интерактивные элементы** — то, с чем взаимодействует пользователь:
   ссылки, кнопки, поля ввода, чекбоксы/радио, селекты, табы, пагинация, иконки-действия
   (профиль, корзина, избранное), элементы карточек («В корзину» и т.п.).
   Для каждого добавь локатор-`@property` → `Locator`.

4. **Не трогать существующие.** Если локатор с таким смыслом уже есть в PageObject —
   НЕ изменяй и НЕ дублируй его. Сопоставляй по назначению, а не только по имени:
   прежде чем добавлять, проверь, не покрыт ли элемент существующим `@property`.
   Добавляй только новые. Существующий код страницы оставляй без изменений.

5. **Отчёт.** В конце выведи таблицу по ДОБАВЛЕННЫМ элементам:
   `имя @property | тип (секция/интерактивный) | селектор | предполагаемая цель`.
   Кратко опиши, для чего, по твоему мнению, каждый элемент будет использоваться
   в тестах. Отдельно перечисли, что было пропущено как уже существующее.

## Ограничения

- PageObject содержит ТОЛЬКО локаторы (`@property` → `Locator`). Никаких действий,
  ожиданий и `expect` — это слой `steps/`.
- Не меняй прод-сайт. Это действие read-only по отношению к странице.
- Не добавляй действия/тесты в рамках этого действия — только локаторы PageObject.
- Именуй `@property` по смыслу элемента (`search_input`, `catalog_link`,
  `add_to_cart_button`, `header_section`, `filters_sidebar`).
